# Exercise - HTTP Content types

Follow me on Twitter, happy to take your suggestions on topics or improvements /Chris

In this exercise, we will show how you can use different content types.

The exercise should show the following things:

  • How to set different content types
  • Learn how to recognize different content type errors and how to fix them
  • Understand how transmitting files differs from sending static content.

# Exercise - setting content types

In this exercise, we will learn how to set different content types. As part of the exercise, we will start by setting a content type and then set the content that matches it. Then we will proceed to set an incorrect content type and learn how a browser responds to that. The reason we show that, is that it's a quite common error to make. Also it's important to recognize the error so you can fix it.

# Setting a content type and content correctly

Start by creating a server-json.js file. Now give the file the following content:

const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'application/json' });
  const products = [{ id: 1, name: 'compass' }, { id: 2, name: 'map' }];
  res.end(JSON.stringify(products));
});

server.listen(8000, () => {
  console.log('server started at 8000');
})

# Setting a content type and content incorrectly

However there are cases when this break. Try changing the Content-Type to application/json and render HTML, like the below code:

res.writeHead(200, { 'Content-Type': 'application/json' });

In conclusion. Do ensure the Content-Type header value matches the actual content being sent. Even though your user might have a competent Browser there might be unexpected effects when rendering.

##Â Exercise - transmitting files

So far we have been transmitting content from static data. What if our data resides in a file? Well, we need to think of resource usage at this point. It's tempting to just open a file and wait for it to finish and then respond with the content. This approach works, for small files. For larger files it will cause a problem. What Node.js does is to put everything in memory, when you open the file. As a result your resource usage will spike. The better way is to use streams. Let me show you how.

Create the file server-streaming.js. Ensure you have an .mp3 file on your computer. Copy said file to the same directory as your server-streaming.js file. Now give your server-streaming.js file the following content:

const http = require('http');
const path = require('path');
const fs = require('fs');

const server= http.createServer((req, res) => {
  const fileStream = fs.createReadStream(path.join(__dirname, 'podcast.mp3'));
  fileStream.pipe(res);
});

server.listen(8000, () => {
  console.log('Running on 8000');
})
